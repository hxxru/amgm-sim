# Project Context Packet for Coding Agents

This file describes what project context a coding agent should receive.

## Give the coding agent

1. This starter repo.
2. The whiteboard sketch in `assets/whiteboard_sketch.jpg`.
3. The files in `docs/`.
4. The first milestone prompt in `prompts/coding_agent_prompt.md`.

## Do not give the coding agent

- The whole AM-GM monograph.
- The whole finite-band typology paper.
- The full internal memo suite.
- The full proof of readout rigidity.
- Philosophical positioning documents.
- Instructions to implement Document 3 estimation.

The agent should build a pedagogical sandbox, not become the theorist.

## Optional excerpts to provide later

If the coding agent needs more context, provide short excerpts only.

Recommended excerpts:

### From Document 1

- scalar compression / rank-one interface explanation;
- weak-coupling Appendix H intuition:
  - leakage relative to compression gap;
  - product/composition integrity;
  - Markov blanket porosity if relevant.

### From Document 2

- signature tuple definition;
- five-region typology summary;
- formal-cone non-diagnostic versus interpretable 2a distinction;
- stratum 3 cone/vector-valued comparison summary;
- dynamic axis summary, if implementing later.

## Context length target

The total context packet should be about 5–10 pages equivalent.

Suggested packet:

```text
1. Design/spec files from this repo
2. AM-GM concept summary
3. Glossary
4. Preset/acceptance criteria
5. Whiteboard image
```
